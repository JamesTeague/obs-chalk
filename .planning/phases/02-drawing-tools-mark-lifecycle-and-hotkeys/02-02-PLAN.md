**ArrowMark (TOOL-02):**
- Constructor: `ArrowMark(float x1, float y1, float x2, float y2, float r, float g, float b, float a = 1.0f)` -- stores tail (x1,y1), head (x2,y2), color as vec4. Initially tail==head (set at mouse-down, head updated on drag).
- `update_end(float x, float y)` override: updates head point (x2_, y2_).
- `draw()`: Draws shaft as GS_LINESTRIP (2 vertices: tail to head). Draws arrowhead as two separate GS_LINESTRIP calls (2 vertices each): from head back along +-30 degrees from the shaft direction. Arrowhead arm length = 15px. Use `std::atan2(y2_-y1_, x2_-x1_)` for shaft angle, then `cos(angle +/- 0.5236f)` and `sin(angle +/- 0.5236f)` for arm endpoints (0.5236 rad = 30 degrees). Skip draw if shaft length < 2px (avoid degenerate arrowhead).
- `distance_to()`: Return minimum distance from query point to the shaft line segment (same point-to-segment formula as FreehandMark, but only one segment).
- Includes: `<cmath>`, `<limits>`, `<graphics/graphics.h>`, `<graphics/vec4.h>`, `"mark.hpp"`

**CircleMark (TOOL-03):**
- Constructor: `CircleMark(float cx, float cy, float r, float red, float g, float b, float a = 1.0f)` -- stores center (cx_,cy_), radius (r_), color as vec4. Initially r_=0 (set at mouse-down with center, radius updated on drag).
- `update_end(float x, float y)` override: updates radius as `std::hypot(x - cx_, y - cy_)`.
- `draw()`: Draws circle outline as GS_LINESTRIP with 64 vertices plus one closing vertex (i from 0 to 64 inclusive). Each vertex: `cx_ + r_ * cos(2*PI*i/64), cy_ + r_ * sin(2*PI*i/64)`. Skip if radius < 1px.
- `distance_to()`: Return `abs(hypot(x - cx_, y - cy_) - r_)` -- distance from query point to the circle ring. This naturally gives 0 for points on the ring and increases with distance from it.
- Includes: `<cmath>`, `<limits>`, `<graphics/graphics.h>`, `<graphics/vec4.h>`, `"mark.hpp"`

**ConeMark (TOOL-04) -- per user decision: filled semi-transparent isosceles triangle. Drag-to-corner geometry, no hardcoded angle.**

Geometry model: The user clicks to place the apex, then drags to place one base corner. The drag endpoint IS corner1. Corner2 is derived by reflection to form a symmetric (isosceles) triangle.

The axis direction is established on the first `update_end` call (the initial drag direction from apex). On subsequent updates, the drag point moves freely -- its projection onto the axis sets the cone depth, and its perpendicular offset from the axis sets the cone half-width. Corner2 mirrors corner1 across the axis. This means the angle is fully determined by where the user drags: dragging along the axis gives a narrow cone, dragging at a steep angle off-axis gives a wide cone. No hardcoded angle.

**Members:**
- `ax_`, `ay_`: apex position (click point)
- `cx_`, `cy_`: dragged corner position (corner1)
- `axis_ux_`, `axis_uy_`: unit axis direction, locked on first update_end
- `axis_set_`: bool, false until first update_end
- `color_`: vec4 with alpha default 0.35f

**Constructor:** `ConeMark(float ax, float ay, float cx, float cy, float r, float g, float b, float a = 0.35f)` -- stores apex (ax_,ay_) and initial corner (cx_,cy_), color as vec4, axis_set_ = false.

**`update_end(float x, float y)` override:**
- If !axis_set_ and distance from apex to (x,y) > 5px: compute axis direction as unit vector from apex toward (x,y). Set `axis_ux_ = dx/len, axis_uy_ = dy/len`. Set `axis_set_ = true`.
- Always update corner: `cx_ = x, cy_ = y`.

**`draw()`:**
1. If !axis_set_, skip (no meaningful triangle yet).
2. Vector from apex to corner1: `dx = cx_ - ax_, dy = cy_ - ay_`. Skip if `hypot(dx,dy) < 2px`.
3. Project corner1 onto the locked axis: `depth = dx*axis_ux_ + dy*axis_uy_`. If depth < 1px, skip (corner is behind or at apex).
4. Perpendicular offset of corner1 from axis: `perp = -dx*axis_uy_ + dy*axis_ux_`. If `abs(perp) < 1px`, use a minimum of 3px to prevent degenerate slivers.
5. Base midpoint M on axis: `mx = ax_ + depth*axis_ux_, my = ay_ + depth*axis_uy_`.
6. Perpendicular unit: `px = -axis_uy_, py = axis_ux_`.
7. Corner1 (actual rendered position): `c1x = mx + perp*px, c1y = mx + perp*py` -- wait, this reconstructs corner1 from the projection. Just use the stored (cx_, cy_) for corner1 directly. No -- use the projection so corner1 is clamped to the base line perpendicular to the axis at depth. Actually, let corner1 = drag point directly, and corner2 = reflection. The base midpoint is the midpoint of corner1 and corner2.
8. Corner2 = reflection of corner1 across the axis line through the apex. To reflect point C across line through A with direction (ux, uy):
   - Translate: `dx = cx_ - ax_, dy = cy_ - ay_`
   - Project onto axis: `proj = dx*axis_ux_ + dy*axis_uy_`
   - Project onto perp: `perp = dx*(-axis_uy_) + dy*axis_ux_`
   - Reflected point: same proj, negated perp
   - `c2x = ax_ + proj*axis_ux_ - perp*(-axis_uy_)` = `ax_ + proj*axis_ux_ + perp*axis_uy_`
   - `c2y = ay_ + proj*axis_uy_ - perp*axis_ux_` = `ay_ + proj*axis_uy_ - perp*axis_ux_`

   Simplified reflection formula:
   - `c2x = 2*ax_ + 2*proj*axis_ux_ - cx_`... let me just write it cleanly:
   - `along = dx*axis_ux_ + dy*axis_uy_` (projection of AC1 onto axis)
   - Point on axis closest to C1: `foot_x = ax_ + along*axis_ux_, foot_y = ay_ + along*axis_uy_`
   - `c2x = 2*foot_x - cx_, c2y = 2*foot_y - cy_` (reflection = 2*foot - original)
9. Call `gs_set_cull_mode(GS_NEITHER)` before drawing.
10. Draw filled triangle: `gs_render_start(false)`, vertices apex, (cx_,cy_), (c2x,c2y), `gs_render_stop(GS_TRIS)`.
11. Restore: `gs_set_cull_mode(GS_BACK)`.

Note: Semi-transparency works automatically -- chalk_video_render already sets `gs_blend_function(GS_BLEND_SRCALPHA, GS_BLEND_INVSRCALPHA)` and the color alpha is 0.35.

**`distance_to()`:**
- Recompute corner2 using the same reflection math as draw().
- Compute min distance from query point to three triangle edges: apex-C1, apex-C2, C1-C2 (point-to-segment formula).
- Also check if point is inside the triangle (cross-product sign test: if all three cross products have same sign, point is inside -> return 0).
- If axis not set, return `std::numeric_limits<float>::max()`.

**Includes:** `<cmath>`, `<limits>`, `<graphics/graphics.h>`, `<graphics/vec4.h>`, `"mark.hpp"`
