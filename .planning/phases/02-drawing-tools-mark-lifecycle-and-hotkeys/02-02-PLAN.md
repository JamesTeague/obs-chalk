**ArrowMark (TOOL-02):**
- Constructor: `ArrowMark(float x1, float y1, float x2, float y2, float r, float g, float b, float a = 1.0f)` -- stores tail (x1,y1), head (x2,y2), color as vec4. Initially tail==head (set at mouse-down, head updated on drag).
- `update_end(float x, float y)` override: updates head point (x2_, y2_).
- `draw()`: Draws shaft as GS_LINESTRIP (2 vertices: tail to head). Draws arrowhead as two separate GS_LINESTRIP calls (2 vertices each): from head back along +-30 degrees from the shaft direction. Arrowhead arm length = 15px. Use `std::atan2(y2_-y1_, x2_-x1_)` for shaft angle, then `cos(angle +/- 0.5236f)` and `sin(angle +/- 0.5236f)` for arm endpoints (0.5236 rad = 30 degrees). Skip draw if shaft length < 2px (avoid degenerate arrowhead).
- `distance_to()`: Return minimum distance from query point to the shaft line segment (same point-to-segment formula as FreehandMark, but only one segment).
- Includes: `<cmath>`, `<limits>`, `<graphics/graphics.h>`, `<graphics/vec4.h>`, `"mark.hpp"`

**CircleMark (TOOL-03):**
- Constructor: `CircleMark(float cx, float cy, float r, float red, float g, float b, float a = 1.0f)` -- stores center (cx_,cy_), radius (r_), color as vec4. Initially r_=0 (set at mouse-down with center, radius updated on drag).
- `update_end(float x, float y)` override: updates radius as `std::hypot(x - cx_, y - cy_)`.
- `draw()`: Draws circle outline as GS_LINESTRIP with 64 vertices plus one closing vertex (i from 0 to 64 inclusive). Each vertex: `cx_ + r_ * cos(2*PI*i/64), cy_ + r_ * sin(2*PI*i/64)`. Skip if radius < 1px.
- `distance_to()`: Return `abs(hypot(x - cx_, y - cy_) - r_)` -- distance from query point to the circle ring. This naturally gives 0 for points on the ring and increases with distance from it.
- Includes: `<cmath>`, `<limits>`, `<graphics/graphics.h>`, `<graphics/vec4.h>`, `"mark.hpp"`

**ConeMark (TOOL-04) -- per user decision: filled semi-transparent isosceles triangle.**

Geometry model (per research recommendation, confirmed by planning context): drag endpoint is the midpoint of the base, half-angle is fixed at ~30 degrees. This is unambiguous and produces the symmetric isosceles triangle the user described.

- Constructor: `ConeMark(float ax, float ay, float tx, float ty, float r, float g, float b, float a = 0.35f)` -- stores apex (ax_,ay_), axis tip / base midpoint (tx_,ty_), color as vec4 with alpha default 0.35 for semi-transparency, and `half_angle_rad_ = M_PI / 6.0f` (30 degrees).
- `update_end(float x, float y)` override: updates base midpoint (tx_, ty_).
- `draw()`:
  1. Compute axis vector: `dx = tx_ - ax_, dy = ty_ - ay_`, length = hypot(dx, dy). Skip if length < 1px.
  2. Unit axis: `ux = dx/len, uy = dy/len`. Perpendicular: `px = -uy, py = ux`.
  3. Half-width at base: `half_w = len * tan(half_angle_rad_)`.
  4. Corner 1: `c1x = tx_ + px*half_w, c1y = ty_ + py*half_w`
  5. Corner 2: `c2x = tx_ - px*half_w, c2y = ty_ - py*half_w`
  6. Call `gs_set_cull_mode(GS_NEITHER)` before drawing to prevent face culling from hiding the triangle when dragged in certain directions.
  7. Draw filled triangle: `gs_render_start(false)`, vertices apex, c1, c2, `gs_render_stop(GS_TRIS)`.
  8. Restore cull mode: `gs_set_cull_mode(GS_BACK)` (OBS default).

  Note: The semi-transparency works automatically because chalk_video_render already sets `gs_blend_function(GS_BLEND_SRCALPHA, GS_BLEND_INVSRCALPHA)` and the color alpha is 0.35.

- `distance_to()`: Compute minimum distance from query point to the three triangle edges (apex-c1, apex-c2, c1-c2). Use the same point-to-segment formula. Need to recompute c1/c2 from stored apex/tip (same math as draw). Return minimum of the three segment distances. Also check if point is INSIDE the triangle -- if so, return 0 (using cross-product sign test: if all three cross products have the same sign, point is inside).

- Includes: `<cmath>`, `<limits>`, `<graphics/graphics.h>`, `<graphics/vec4.h>`, `"mark.hpp"`
