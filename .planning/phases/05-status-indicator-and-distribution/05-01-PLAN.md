<task type="auto">
  <name>Task 1: Create ChalkDock widget with QTimer polling</name>
  <files>src/chalk-dock.hpp, src/chalk-dock.cpp</files>
  <action>
Create the dock panel widget as two new files:

**chalk-dock.hpp:**
- `#pragma once`, include QWidget, QLabel forward declares
- `class ChalkDock : public QWidget` with `Q_OBJECT`
- Public: constructor, no other public API needed (self-updating via timer)
- Private: `QLabel *mode_label_`, `QLabel *tool_label_`, `QLabel *color_swatch_`, `QLabel *color_name_label_`
- Private slot: `void refresh()`

**chalk-dock.cpp:**
- Include chalk-dock.hpp, chalk-source.hpp, tool-state.hpp, obs-frontend-api.h
- Include QVBoxLayout, QHBoxLayout, QLabel, QTimer, QFont
- Constructor:
  - Create a QVBoxLayout
  - Row 1: "Mode:" label + `mode_label_` (bold text, shows "ACTIVE" or "off")
  - Row 2: "Tool:" label + `tool_label_` (shows tool name string)
  - Row 3: "Color:" label + `color_swatch_` (fixed-size label with background color) + `color_name_label_`
  - `addStretch()` at bottom
  - Set minimum width ~180px so labels aren't clipped
  - Create a `QTimer` with 100ms interval, connect timeout to `refresh()`, start
- `refresh()` slot:
  - Call `chalk_mode_is_active()` (new accessor, see Task 2) to update mode label
  - Call `chalk_find_source()` to get ChalkSource*
  - If nullptr: show "(no source)" for tool, gray swatch for color, return
  - Read `ctx->tool_state.active_tool` and map to string via a static `tool_name()` function (switch on ToolType enum, return const char*)
  - Read `ctx->tool_state.color_index` and `ctx->tool_state.active_color()`:
    - Convert ABGR to CSS #RRGGBB: `r = abgr & 0xFF`, `g = (abgr >> 8) & 0xFF`, `b = (abgr >> 16) & 0xFF`
    - Set `color_swatch_` background via setStyleSheet; use black or white text depending on brightness
    - Map color_index to name via static `color_name()` function (array: "White", "Yellow", "Red", "Blue", "Green")
- Include `"chalk-dock.moc"` at the bottom of the .cpp (same AUTOMOC pattern as chalk-mode.cpp, since Q_OBJECT is in the header this may not be needed -- only include the .moc if the class is defined in the .cpp; since the class is in the .hpp, AUTOMOC will handle it automatically via the header, so do NOT include a .moc line)

**Thread safety note:** The QTimer fires on the Qt main thread. `chalk_find_source()` accesses OBS scene state which is safe from the Qt main thread (same thread context as all existing callers). Reading `tool_state` fields (active_tool, color_index) is safe without mutex -- they are only written by hotkey callbacks which are serialized by OBS, and a torn read of an int or uint8_t on x86/ARM is not possible.

**Anti-patterns to avoid:**
- Do NOT use `obs_frontend_add_dock()` (deprecated) -- that's called from chalk-mode.cpp, not here
- Do NOT delete the dock widget manually -- OBS owns it after registration
- Do NOT use signals/slots to chalk-source.cpp -- polling is simpler and avoids cross-file coupling
