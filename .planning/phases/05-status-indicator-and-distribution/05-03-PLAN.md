<objective>
Fix Windows preview interaction by adding a QAbstractNativeEventFilter that intercepts Win32 mouse messages on the preview HWND, and clean up debug diagnostic logging from the investigation commits.

Purpose: The Windows DLL loads and chalk mode activates (crosshair cursor appears), but clicking the preview creates OBS's selection box instead of drawing. This is because Qt eventFilter never receives mouse events on Windows when WA_NativeWindow creates a separate HWND. The native event filter intercepts Win32 messages before Qt dispatches them, making drawing work on Windows.
Output: Modified `chalk-mode.cpp` with platform-conditional native event filter and cleaned-up logging.

**2. Add file-scope state for the native filter (in the file-scope state section, guarded):**
```cpp

class ChalkNativeFilter;
static ChalkNativeFilter *s_native_filter = nullptr;
static HWND               s_preview_hwnd  = nullptr;
#endif
```

**3. Define ChalkNativeFilter class (after the file-scope state section, before ChalkEventFilter, guarded):**
```cpp
#ifdef _WIN32
class ChalkNativeFilter : public QAbstractNativeEventFilter {
public:
    bool nativeEventFilter(const QByteArray &eventType,
                           void *message,
                           qintptr *result) override
    {
        if (eventType != "windows_generic_MSG")
            return false;

        auto *msg = static_cast<MSG *>(message);

        // Only intercept messages targeted at the preview HWND
        if (msg->hwnd != s_preview_hwnd)
            return false;

        // Only intercept when chalk mode is active
        if (!s_chalk_mode_active)
            return false;

        switch (msg->message) {
        case WM_LBUTTONDOWN: {
            // lParam contains client coordinates in low/high words
            float client_x = static_cast<float>(static_cast<short>(LOWORD(msg->lParam)));
            float client_y = static_cast<float>(static_cast<short>(HIWORD(msg->lParam)));
            // Convert client pixels to Qt logical pixels (devicePixelRatio handled inside)
            float dpr = static_cast<float>(s_preview->devicePixelRatioF());
            float logical_x = client_x / dpr;
            float logical_y = client_y / dpr;
            vec2 pos = preview_widget_to_scene(s_preview, logical_x, logical_y);
            ChalkSource *ctx = chalk_find_source();
            if (ctx) chalk_input_begin(ctx, pos.x, pos.y);
            *result = 0;
            return true;
        }
        case WM_MOUSEMOVE: {
            // Only intercept if left button is down (wParam MK_LBUTTON)
            if (!(msg->wParam & MK_LBUTTON))
                return false;
            float client_x = static_cast<float>(static_cast<short>(LOWORD(msg->lParam)));
            float client_y = static_cast<float>(static_cast<short>(HIWORD(msg->lParam)));
            float dpr = static_cast<float>(s_preview->devicePixelRatioF());
            float logical_x = client_x / dpr;
            float logical_y = client_y / dpr;
            vec2 pos = preview_widget_to_scene(s_preview, logical_x, logical_y);
            ChalkSource *ctx = chalk_find_source();
            if (ctx) chalk_input_move(ctx, pos.x, pos.y);
            *result = 0;
            return true;
        }
        case WM_LBUTTONUP: {
            ChalkSource *ctx = chalk_find_source();
            if (ctx) chalk_input_end(ctx);
            *result = 0;
            return true;
        }
        default:
            return false;
        }
    }
};
#endif
```

**Key design decisions:**
- `eventType == "windows_generic_MSG"` is how Qt6 identifies Win32 messages in QAbstractNativeEventFilter. This is the documented eventType string.
- Client coordinates from lParam are in physical pixels (DPI-scaled on Windows). Divide by devicePixelRatio to get logical pixels before passing to `preview_widget_to_scene()`, which expects logical pixel input (see its implementation: it multiplies by dpr internally).
  - IMPORTANT: Actually re-read `preview_widget_to_scene` -- it takes logical pixels and immediately multiplies by dpr to get physical pixels. Win32 lParam gives physical pixels directly. So we need to divide by dpr to convert physical->logical BEFORE calling the function, which then multiplies back to physical internally. This is correct.
  - ALTERNATIVE: If coordinate mapping is off on Windows, the issue may be that Win32 client coords are already in physical pixels and `preview_widget_to_scene` expects logical. The dpr division handles this. If testing reveals coordinates are wrong, the executor should check whether lParam coords on the streaming machine's DPI setting need different handling.
- `LOWORD/HIWORD` cast through `short` to handle negative coordinates (when mouse is dragged outside the window).
- Only WM_LBUTTONDOWN, WM_MOUSEMOVE (with MK_LBUTTON), WM_LBUTTONUP are intercepted. Right-click and all other messages pass through.
- Tablet/pen (WM_POINTER) is explicitly deferred per CONTEXT.md.
