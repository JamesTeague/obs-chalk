  <action>
This task pushes the current main branch (which now includes the dock panel from Plan 01) to the GitHub remote, then creates and pushes a semver tag to trigger the full CI pipeline.

Steps:
1. Ensure all code is committed and the working tree is clean
2. Push main branch to origin: `git push origin main`
3. Create annotated tag: `git tag -a 0.1.0 -m "v0.1.0: initial release with dock panel, all drawing tools, and chalk mode"`
4. Push tag: `git push origin 0.1.0`
5. Monitor CI run: `gh run list --limit 5` to confirm the push workflow started

The tag name `0.1.0` matches the version in `buildspec.json`. The push.yaml workflow detects the semver tag pattern and sets `notarize:true` and `config:Release`, which triggers the full codesign + notarize + package pipeline.

**Important:** If Apple Developer secrets are not yet configured in GitHub repo settings, the macOS codesign step will be skipped (the CI handles this gracefully -- it checks `fromJSON(steps.codesign.outputs.haveCodesignIdent)`). The Windows build does not require any secrets and will produce artifacts regardless.

**If push fails (no remote configured):** Create the remote first with `gh repo create` or `git remote add origin <url>`. This is the collaborative fallback point.
