<task type="auto">
  <name>Task 2: Create chalk-mode module with event filter, coordinate mapping, hotkey, and cursor</name>
  <files>src/chalk-mode.hpp, src/chalk-mode.cpp, src/plugin.cpp</files>
  <action>
**src/chalk-mode.hpp:** Create the public API for chalk mode:

```cpp
#pragma once
// Lifecycle: called from plugin.cpp frontend event callback
void chalk_mode_install();   // Called on OBS_FRONTEND_EVENT_FINISHED_LOADING
void chalk_mode_shutdown();  // Called on OBS_FRONTEND_EVENT_EXIT
```

The `ChalkEventFilter` class is internal to chalk-mode.cpp (not exposed in header). It inherits `QObject` and uses `Q_OBJECT` macro for MOC.

**src/chalk-mode.cpp:** Implement the full chalk mode system:

1. **File-scope state:**
   - `static QWidget *s_preview = nullptr;`
   - `static ChalkEventFilter *s_filter = nullptr;`
   - `static obs_hotkey_id s_chalk_mode_hotkey = OBS_INVALID_HOTKEY_ID;`
   - `static bool s_chalk_mode_active = false;`

2. **Coordinate mapping** — `preview_widget_to_scene(QWidget *preview, float event_x, float event_y) -> vec2`:
   - Get `devicePixelRatioF()` from preview widget
   - Get physical pixel size: `width * dpr`, `height * dpr`
   - Get OBS canvas dimensions via `obs_get_video_info()`
   - Compute letterbox offset and scale: inner dimensions = physical - 2*EDGE (EDGE=10), then aspect-ratio comparison to get scale and centering offsets (reimplemented `GetScaleAndCenterPos` — see research Pattern 3 for exact formula)
   - Convert event pos (logical px) to scene px: `scene_x = (event_x * dpr - off_x) / scale`
   - Add `// TODO: fixed-scale mode` comment

3. **ChalkEventFilter class** (Q_OBJECT, defined in .cpp):
   - `eventFilter(QObject*, QEvent*)` override:
     - If `!s_chalk_mode_active`, return false (pass through)
     - Switch on event type:
       - `QEvent::TabletPress`: call `handle_tablet` -> `chalk_input_begin` (pressure stored for Plan 02)
       - `QEvent::TabletMove`: call `handle_tablet` -> `chalk_input_move`
       - `QEvent::TabletRelease`: call `handle_tablet` -> `chalk_input_end`
       - `QEvent::MouseButtonPress` (left button only): map coords -> `chalk_input_begin`, return true
       - `QEvent::MouseMove` (with left button held): map coords -> `chalk_input_move`, return true
       - `QEvent::MouseButtonRelease` (left button): `chalk_input_end`, return true
       - Default: return false
     - For tablet events, return true always to prevent Qt mouse synthesis (Pitfall 3)
   - `handle_tablet(QTabletEvent*)`: extract `position()` (NOT deprecated `pos()`), call `preview_widget_to_scene`, dispatch to chalk_input. For now, ignore pressure (Plan 02 adds that). Return true.

4. **Global hotkey:**
   - Register in `chalk_mode_install` using `obs_hotkey_register_frontend("chalk.chalk_mode", "Chalk: Toggle Chalk Mode", ...)`.
   - Callback toggles `s_chalk_mode_active` and calls cursor update.
   - NOTE: Research says register in obs_module_load, but the research also says FINISHED_LOADING is too late for *restoring saved bindings*. Since this is a global hotkey (not source-scoped), register it in `chalk_mode_install` which fires at FINISHED_LOADING. If binding persistence is an issue, this will surface during verification and can be moved to obs_module_load.

5. **Cursor toggle:**
   - When chalk mode activates: `s_preview->setCursor(Qt::CrossCursor)`
   - When chalk mode deactivates: `s_preview->unsetCursor()`

6. **chalk_mode_install():** Get main window via `obs_frontend_get_main_window()`, cast to `QMainWindow*`, `findChild<QWidget*>("preview")`. If null, log warning and return (graceful degradation per discretion). Create `ChalkEventFilter`, install on preview. Register global hotkey.

7. **chalk_mode_shutdown():** Remove event filter, delete filter, unregister hotkey. Null out statics.

**src/plugin.cpp:** Add frontend event callback:
- `#include <obs-frontend-api.h>` and `#include "chalk-mode.hpp"`
- In `obs_module_load()`: add `obs_frontend_add_event_callback(on_obs_event, nullptr);` after source registration.
- Static callback: on `OBS_FRONTEND_EVENT_FINISHED_LOADING` call `chalk_mode_install()`, on `OBS_FRONTEND_EVENT_EXIT` call `chalk_mode_shutdown()`.
- In `obs_module_unload()`: remove frontend event callback.

IMPORTANT: Because chalk-mode.cpp uses Q_OBJECT macro, AUTOMOC must process it. The CMake AUTOMOC property is already set when ENABLE_QT=ON. However, Q_OBJECT in a .cpp file requires a `#include "chalk-mode.moc"` at the END of chalk-mode.cpp for MOC to find it (since the class is not in a header).
