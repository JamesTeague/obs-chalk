<objective>
Add tablet pressure sensitivity to freehand strokes so pen pressure varies stroke width continuously.

Purpose: INPT-03 — the core value for tablet/pen users. Without pressure sensitivity, tablet input is no different from a mouse. Variable-width strokes give the coach natural pen feel during film review.

Output: FreehandMark stores per-point pressure, renders as variable-width quad strip instead of hairline LINESTRIP. Event filter passes QTabletEvent pressure through the input dispatch chain. Mouse input defaults to pressure=1.0 (full width).
</objective>

**src/marks/mark.hpp:** Change `add_point` signature to include pressure:
```cpp
virtual void add_point(float x, float y, float pressure = 1.0f) {}
```
Default parameter means existing callers (if any) are unaffected. No other mark types override `add_point`, so this is safe.

**src/marks/freehand-mark.hpp:**
1. Add pressure to `FreehandPoint`:
```cpp
struct FreehandPoint {
    float x;
    float y;
    float pressure;  // 0.0-1.0, where 1.0 = full width
};
```
2. Update `add_point` override signature:
```cpp
void add_point(float x, float y, float pressure = 1.0f) override;
```
3. Add a constant for stroke width range:
```cpp
static constexpr float CHALK_MIN_WIDTH = 1.5f;   // hairline at zero pressure
static constexpr float CHALK_MAX_WIDTH = 6.0f;    // full width at max pressure
```

**src/marks/freehand-mark.cpp:**
1. Update `add_point` to store pressure on each point. Interpolated points (the STEP=2 loop) should lerp pressure between the previous point and the new point.
2. Update `draw()` to render as a variable-width quad strip instead of LINESTRIP:
   - For each segment between consecutive points, compute the width at each endpoint: `width = CHALK_MIN_WIDTH + pressure * (CHALK_MAX_WIDTH - CHALK_MIN_WIDTH)`
   - Compute the perpendicular offset at each point (direction perpendicular to the segment, scaled by half-width)
   - Emit two vertices per point (one on each side of the stroke centerline) into a `GS_TRISTRIP`
   - This replaces the current `GS_LINESTRIP` approach entirely
   - For single-point strokes or very short strokes, fall back to the LINESTRIP rendering (or a dot)
3. `distance_to` — no change needed, centerline distance is still correct for hit-testing.

**src/chalk-source.hpp:** Update shared dispatch signatures to include pressure:
```cpp
void chalk_input_begin(ChalkSource *ctx, float x, float y, float pressure = 1.0f);
void chalk_input_move(ChalkSource *ctx, float x, float y, float pressure = 1.0f);
```
(`chalk_input_end` does not need pressure.)

**src/chalk-source.cpp:**
1. Update `chalk_input_begin` implementation: pass pressure to `mark->add_point(x, y, pressure)` for freehand marks. Other tool types ignore pressure (arrows, circles, cones have uniform width).
2. Update `chalk_input_move` implementation: pass pressure to `in_progress->add_point(x, y, pressure)` for freehand.
3. Source callbacks (`chalk_mouse_click`, `chalk_mouse_move`) call dispatch with default pressure=1.0 (mouse has no pressure).

  <name>Task 2: Wire tablet pressure from event filter through dispatch</name>

  <action>
Update the tablet event handling in `ChalkEventFilter::handle_tablet` (created in Plan 01) to extract and pass pressure:

1. In `handle_tablet(QTabletEvent *e)`:
   - Extract pressure: `float pressure = static_cast<float>(e->pressure());` — returns 0.0-1.0
   - Extract position: `e->position().x()`, `e->position().y()` (NOT deprecated `pos()`)
   - Map to scene coords via `preview_widget_to_scene`
   - For `QEvent::TabletPress`: call `chalk_input_begin(ctx, scene.x, scene.y, pressure)`
   - For `QEvent::TabletMove`: call `chalk_input_move(ctx, scene.x, scene.y, pressure)`
   - For `QEvent::TabletRelease`: call `chalk_input_end(ctx)`
   - Return true for all tablet events (prevents Qt mouse synthesis — Pitfall 3)

2. Verify that mouse event handling in the filter passes pressure=1.0 (the default parameter value). No code change needed if the default is used, but add a comment: `// Mouse has no pressure — uses default 1.0f`.

3. Edge case: if a tablet event has pressure exactly 0.0 (pen hovering but not touching), treat TabletPress with pressure < 0.01 as no-op (don't start a mark with invisible width). This prevents ghost strokes from hover events that some tablets report as pressure-0 press events.
