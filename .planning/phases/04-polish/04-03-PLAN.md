  <name>Task 2: Verify all Phase 4 behaviors in OBS</name>
  <action>
All five Phase 4 behavior changes have been implemented: consistent 6px line width on all mark types, transparent cone fill, persistent delete mode, laser-as-tool, and clear-on-scene-transition setting. Human verification is required to confirm visual and interactive correctness in a live OBS session.
  </action>
  <how-to-verify>
Launch OBS and add an obs-chalk source to a scene. Then test each behavior:

**1. UX-01 — Line width consistency:**
- Select Arrow tool (hotkey), draw an arrow
- Select Circle tool (hotkey), draw a circle
- Select Cone tool (hotkey), draw a cone
- Select Freehand tool (hotkey), draw a freehand stroke with mouse (no tablet)
- Verify: All four mark types have visually similar ~6px stroke weight. Arrow shaft, arrowhead arms, circle outline, and cone side edges should match freehand thickness.

**2. UX-02 — Cone transparency:**
- Draw a cone over existing marks or scene content
- Verify: The cone interior is visibly transparent — you can see content through the cone fill. The fill should not be fully opaque.

**3. UX-03 — Persistent delete mode:**
- Draw 3+ marks of any type
- Press the pick-to-delete hotkey to enter delete mode
- Click on one mark — it should be removed
- WITHOUT pressing the hotkey again, click on another mark — it should also be removed
- Verify: Delete mode stays active across multiple clicks until you press the toggle hotkey again.

**4. UX-04 — Laser as selectable tool:**
- Select the Laser tool via hotkey (Chalk: Laser in OBS hotkey settings)
- With chalk mode active, press and hold the mouse button — laser dot should appear
- Move the mouse while holding — dot follows cursor
- Release the mouse button — dot disappears
- Verify: Laser shows on mouse-down, hides on mouse-up. Laser is selected via tool hotkey, not a hold-key.

**5. INTG-01 — Clear on scene transition:**
- Right-click the obs-chalk source > Properties
- Verify: "ClearOnSceneChange" checkbox is visible (should be unchecked by default)
- Draw some marks, switch to a different scene and back — marks should persist (setting is off)
- Enable the checkbox in Properties, draw marks, switch scenes
- Verify: Marks are cleared when transitioning to the scene with the setting enabled
  </how-to-verify>
