  <action>
Replace the three GS_LINESTRIP render calls (shaft, left arm, right arm) with GS_TRISTRIP using perpendicular offset vertices at const HALF_W = 3.0f (6px total width).

For each line segment (P1 to P2):
1. Compute direction: dx = P2.x - P1.x, dy = P2.y - P1.y
2. Normalize: len = hypot(dx, dy); guard len < 1e-6f
3. Perpendicular unit: px = -dy/len, py = dx/len
4. Emit 4 vertices as GS_TRISTRIP:
   - P1 + perp * HALF_W
   - P1 - perp * HALF_W
   - P2 + perp * HALF_W
   - P2 - perp * HALF_W

Apply this pattern three times in draw():
- Shaft: (x1_, y1_) to (x2_, y2_) — one gs_render_start/stop block
- Left arrowhead arm: (x2_, y2_) to (la_x, la_y) — one gs_render_start/stop block
- Right arrowhead arm: (x2_, y2_) to (ra_x, ra_y) — one gs_render_start/stop block

Keep the early return for len < 2.0f, keep the arrowhead angle math (ARM_LEN=15.0f, ARM_ANGLE=0.5236f) unchanged. Only the rendering primitives change.

Add a local helper at file scope to avoid repeating the 4-vertex tristrip pattern three times:

```cpp
static void draw_thick_segment(float x1, float y1, float x2, float y2, float half_w)
{
    float dx = x2 - x1, dy = y2 - y1;
    float len = std::hypot(dx, dy);
    if (len < 1e-6f) return;
    float px = -dy / len, py = dx / len;
    gs_render_start(false);
    gs_vertex2f(x1 + px * half_w, y1 + py * half_w);
    gs_vertex2f(x1 - px * half_w, y1 - py * half_w);
    gs_vertex2f(x2 + px * half_w, y2 + py * half_w);
    gs_vertex2f(x2 - px * half_w, y2 - py * half_w);
    gs_render_stop(GS_TRISTRIP);
}
```

Then draw() becomes:
```cpp
gs_effect_set_vec4(color_param, &color_);
const float HALF_W = 3.0f;
draw_thick_segment(x1_, y1_, x2_, y2_, HALF_W);
// arrowhead arms (same angle math as before)
draw_thick_segment(x2_, y2_, la_x, la_y, HALF_W);
draw_thick_segment(x2_, y2_, ra_x, ra_y, HALF_W);
```

Do NOT change distance_to or update_end — only draw() changes.

**CircleMark (circle-mark.cpp):**

Replace the GS_LINESTRIP circle loop with a GS_TRISTRIP quad strip using radial normal offsets.

The existing loop runs i=0 to i=CIRCLE_PTS (64 iterations). For each point on the circle, the outward radial direction IS the perpendicular to the tangent. So:

```cpp
const float HALF_W = 3.0f;
gs_render_start(false);
for (int i = 0; i <= CIRCLE_PTS; ++i) {
    float angle = TWO_PI * static_cast<float>(i) / static_cast<float>(CIRCLE_PTS);
    float cos_a = std::cos(angle), sin_a = std::sin(angle);
    float cx_pt = cx_ + r_ * cos_a;
    float cy_pt = cy_ + r_ * sin_a;
    // Outward radial normal = (cos_a, sin_a)
    gs_vertex2f(cx_pt + cos_a * HALF_W, cy_pt + sin_a * HALF_W);
    gs_vertex2f(cx_pt - cos_a * HALF_W, cy_pt - sin_a * HALF_W);
}
gs_render_stop(GS_TRISTRIP);
```

This produces 130 vertices (65 iterations x 2). The loop closing at i=CIRCLE_PTS wraps back to angle=0, closing the strip seamlessly.

Do NOT change distance_to or update_end.
